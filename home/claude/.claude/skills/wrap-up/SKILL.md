Review this session before closing. Work through each checkpoint and report the result concisely:

1. **Open topics** — scan the conversation for questions, issues, or requests that were raised but not resolved. List any that are still open; confirm if all are done.

2. **Memory** — identify any insights, decisions, user preferences, or non-obvious context from this session worth persisting. Save any missing ones now using the auto-memory system.

3. **Future plans** — check if any follow-up actions, TODOs, or deferred decisions were mentioned. Confirm they are captured in memory, a CLAUDE.md, or explicitly handed off to the user.

4. **Delivery** — confirm the core task worked on this session is complete and functional. If not, describe what is missing.

5. **Git** — run `git status` in the working directory. If there are uncommitted changes, list them and ask the user whether to commit.

Report each item with a clear status (done / attention needed). Be direct and brief.
